# MigrationToExtension

Инструмент для автоматического переноса доработок из доработанной типовой конфигурации 1С:Предприятие в расширение конфигурации (CFE).

Вместо ручного создания расширения и построчного сравнения кода — автоматизированный пайплайн, который анализирует различия между типовой и доработанной конфигурациями, извлекает изменённые процедуры, генерирует код расширения с diff-маркерами `#Вставка`/`#Удаление` и собирает готовую структуру расширения с XML-метаданными.

## Зачем это нужно

При переходе с доработанной типовой конфигурации на расширения нужно:
1. Найти все изменённые процедуры в десятках модулей
2. Понять, что именно изменено в каждой процедуре
3. Оформить каждое изменение как перехват `&ИзменениеИКонтроль` с корректными маркерами
4. Сгенерировать XML-метаданные расширения

Делать это вручную для конфигурации с сотнями доработок — недели работы. Этот инструмент выполняет это за минуты.

## Требования

- **Python 3.10+** (используется только стандартная библиотека, без внешних зависимостей)
- **PowerShell 5.1+** (для скиллов работы с XML-метаданными 1С)
- **1С:Предприятие** — для выгрузки конфигураций в файлы и создания отчёта о сравнении
- **Claude Code** или **Cursor с AI-агентом** (опционально, для ИИ-ревью крупных блоков изменений)

## Входные данные

Для работы инструмента нужно подготовить три вещи:

| Что | Как получить |
|-----|-------------|
| **Выгрузка доработанной конфигурации** | Конфигуратор 1С → Конфигурация → Выгрузить конфигурацию в файлы |
| **Выгрузка типовой конфигурации** | Аналогично, из чистой типовой конфигурации того же релиза |
| **Отчёт о сравнении** (`ОтчетОСравнении.txt`) | Конфигуратор → Конфигурация → Сравнить конфигурации → Подробный отчёт → Сохранить в файл |

## Быстрый старт

### 1. Клонировать репозиторий

```bash
git clone https://github.com/<your-username>/MigrationToExtension.git
cd MigrationToExtension
```

### 2. Создать конфигурационный файл

Создайте `config.json` в корне проекта:

```json
{
    "customConfigPath": "C:/path/to/modified-config",
    "baseConfigPath": "C:/path/to/base-config",
    "extensionOutputPath": "extension",
    "workDir": "work",
    "extensionName": "МоёРасширение",
    "extensionPrefix": "МР",
    "extensionPurpose": "Customization",
    "compatibilityMode": "Version8_3_24"
}
```

| Параметр | Описание |
|----------|----------|
| `customConfigPath` | Путь к папке с выгрузкой доработанной конфигурации |
| `baseConfigPath` | Путь к папке с выгрузкой типовой (исходной) конфигурации |
| `extensionOutputPath` | Папка для результата — файлов расширения (по умолчанию `extension`) |
| `workDir` | Рабочая директория для промежуточных файлов (по умолчанию `work`) |
| `extensionName` | Имя расширения (для `Configuration.xml`) |
| `extensionPrefix` | Префикс для переименования процедур (напр. `ЕТ` → `ЕТ_ИмяПроцедуры`) |
| `extensionPurpose` | Назначение: `Customization`, `Patch` или `AddOn` |
| `compatibilityMode` | Режим совместимости 1С (напр. `Version8_3_24`) |

### 3. Подготовить отчёт о сравнении

Скопируйте файл подробного отчёта о сравнении конфигураций в корень проекта под именем `ОтчетОСравнении.txt`.

### 4. Запустить пайплайн

Пайплайн выполняется пошагово. Каждый шаг — отдельный скрипт, который можно запустить независимо.

Достаточно просто написать промт: `Выполни перенос всех доработок в расширение` и весь пайплайн будет выполнен последовательно

```bash
# Шаг 1: Сокращение отчёта о сравнении → компактный реестр изменений
python scripts/reduce-comparison-report.py "ОтчетОСравнении.txt" -o "РеестрИзменений.txt"

# Шаг 2: Извлечение процедур из модулей в отдельные файлы
python scripts/extract-modified-procedures.py "РеестрИзменений.txt" -c config.json -o "РеестрПроцедур.txt"

# Шаг 3: Генерация кода расширения с diff-маркерами
python scripts/generate-extension-procedures.py work -c config.json --report "ОтчётГенерации.txt"

# Шаг 4: Сборка отдельных процедур в модули расширения
python scripts/assemble-extension-modules.py work -c config.json --report "ОтчётСборки.txt"

# Шаг 5: Верификация собранных модулей
python scripts/verify-extension-modules.py work -c config.json --report "ОтчётВерификации.txt"

# Шаг 6: Развёртывание модулей в каталог расширения
python scripts/deploy-extension-modules.py work -c config.json --report "ОтчётДеплоя.txt"

# Шаг 7: Генерация XML-метаданных расширения
python scripts/generate-extension-metadata.py work -c config.json --report "ОтчётМетаданных.txt"
```

## Пайплайн

```
ОтчетОСравнении.txt
        │
        ▼
 [1] reduce-comparison-report.py
        │
        ▼
 РеестрИзменений.txt
        │
        ▼
 [2] extract-modified-procedures.py
        │
        ▼
 work/*_typ.bsl + *_mod.bsl + *_new.bsl
        │
        ▼
 [3] generate-extension-procedures.py
        │
        ▼
 work/*_ext.bsl
        │
        ▼
 [4] assemble-extension-modules.py
        │
        ▼
 work/*_module.bsl
        │
        ▼
 [5] verify-extension-modules.py
        │
        ▼
 [6] deploy-extension-modules.py
        │
        ▼
 [7] generate-extension-metadata.py
        │
        ▼
 extension/   ← готовое расширение
```

### Описание шагов

| # | Скрипт | Что делает |
|---|--------|-----------|
| 1 | `reduce-comparison-report.py` | Парсит подробный отчёт 1С о сравнении → компактный реестр модулей с номерами строк |
| 2 | `extract-modified-procedures.py` | По реестру извлекает процедуры из BSL-модулей в отдельные файлы (`_typ` — типовая, `_mod` — доработанная, `_new` — новая) |
| 3 | `generate-extension-procedures.py` | Сравнивает `_typ` vs `_mod` через difflib → генерирует `_ext.bsl` с маркерами `#Вставка`/`#Удаление`, проверяет инвариант |
| 4 | `assemble-extension-modules.py` | Собирает `_ext` + `_new` в `_module.bsl`, переименовывает новые процедуры с префиксом расширения |
| 5 | `verify-extension-modules.py` | Верификация: инвариант, префикс, орфанные ссылки, структура BSL |
| 6 | `deploy-extension-modules.py` | Копирует `_module.bsl` в структуру каталогов расширения |
| 7 | `generate-extension-metadata.py` | Генерирует XML-метаданные: `Configuration.xml`, объекты, формы, роль, язык |

### Опциональный шаг: ИИ-ревью

После шага 3, если в `ОтчётГенерации.txt` отмечены процедуры с крупными блоками замен (`needs_review: true`), можно запустить ИИ-ревью через Claude Code или Cursor. ИИ-агент разбивает крупные `#Вставка`/`#Удаление` блоки на точечные изменения, минимизируя конфликты при обновлении типовой.

Для этого используется скилл `/migrate-to-extension` (см. раздел «Работа с ИИ-агентом»).

## Декораторы расширений 1С

Инструмент генерирует код расширения со следующими декораторами:

| Декоратор | Когда используется |
|-----------|-------------------|
| `&ИзменениеИКонтроль("ИмяПроцедуры")` | Процедура изменена — полная копия с diff-маркерами |
| `&Перед("ИмяПроцедуры")` | Код выполняется до оригинальной процедуры |
| `&После("ИмяПроцедуры")` | Код выполняется после оригинальной процедуры |
| `&Вместо("ИмяПроцедуры")` | Полная замена оригинальной процедуры |

### Diff-маркеры в коде расширения

```bsl
&ИзменениеИКонтроль("ОригинальнаяПроцедура")
Процедура Расш_ОригинальнаяПроцедура(Параметры)
    // Неизменённый типовой код — без маркеров
    ТиповойКод();

#Вставка
    НовыйДобавленныйКод();
#КонецВставки

#Удаление
    УдалённыйТиповойКод();
#КонецУдаления

    // Ещё типовой код без изменений
КонецПроцедуры
```

**Инвариант:** если убрать блоки `#Вставка...#КонецВставки` с содержимым и аннотации `#Удаление`/`#КонецУдаления` (оставив содержимое) — должен получиться исходный типовой код.

## Работа с ИИ-агентом

Проект включает набор скиллов для работы с Claude Code или Cursor AI. Скиллы находятся в `.claude/skills/` и предоставляют команды:

| Команда | Описание |
|---------|----------|
| `/migrate-to-extension` | Полный пайплайн миграции (все шаги) |
| `/cfe-init` | Создать scaffold расширения |
| `/cfe-borrow` | Заимствовать объект из конфигурации |
| `/cfe-patch-method` | Перехватить метод: `&Перед`/`&После`/`&ИзменениеИКонтроль` |
| `/cfe-diff` | Анализ состава расширения |
| `/inspect` | Инспекция объекта 1С (конфигурация, метаданные, форма, СКД и др.) |
| `/validate` | Валидация объекта 1С |

Также доступны workflow-скиллы: `/brainstorm`, `/write-plan`, `/subagent-dev`, `/session-save`, `/session-restore`.

Спецификации XML-форматов 1С (`1c-extension-spec.md`, `1c-configuration-spec.md`, `1c-config-objects-spec.md`) доступны в `.claude/docs/` и используются скиллами при генерации метаданных.
