# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

- Создание новых папок и файлов — без подтверждения

- Запуск `uv run python` — без подтверждения

## Project Purpose

Инструмент для переноса доработок из доработанной типовой конфигурации 1С в расширение конфигурации.

### Входные данные
- **Доработанная конфигурация** — папка с выгрузкой файлов доработанной конфигурации 1С
- **Типовая конфигурация** — папка с выгрузкой файлов исходной (типовой) конфигурации
- **Реестр доработок** — файл со списком доработанных процедур по каждому модулю

### Выходные данные
- **Расширение конфигурации** — третья папка с файлами расширения, содержащими только доработанные процедуры с аннотациями

## Architecture

### Пайплайн обработки

```
ОтчётОСравнении.txt → [1] reduce → РеестрИзменений.txt → [2] extract → work/*_{typ,mod,new}.bsl → [3] generate → *_ext.bsl → [4] assemble → _module.bsl → [5] verify → [6] deploy → [7] metadata → extension/
```

1. **reduce-comparison-report.py** — парсит подробный отчёт 1С о сравнении → компактный реестр модулей с номерами строк
2. **extract-modified-procedures.py** — по реестру извлекает процедуры из BSL-модулей в отдельные файлы (`_typ` + `_mod` для изменённых, `_new` для новых)
3. **generate-extension-procedures.py** — сравнивает `_typ` vs `_mod` через difflib → генерирует `_ext.bsl` с маркерами `#Вставка/#Удаление`, проверяет инвариант
4. **assemble-extension-modules.py** — собирает `_ext` + `_new` в `_module.bsl`, переименование с префиксом
5. **verify-extension-modules.py** — верификация: инвариант, префикс, орфанные ссылки, структура BSL
6. **deploy-extension-modules.py** — копирует `_module.bsl` в структуру расширения
7. **generate-extension-metadata.py** — генерирует XML-метаданные расширения (Configuration.xml, объекты, формы)

> Подробное описание каждого скрипта: см. auto-memory `pipeline.md`

### Ключевые особенности
- Модули могут содержать до 40 000 строк — прямая передача в контекст нецелесообразна
- Процедуры выделяются в отдельные файлы перед анализом для экономии контекста
- Язык исходного кода: 1С (BSL — Built-in Scripting Language, файлы `.bsl`)

### Декораторы перехвата расширений

| Декоратор | Описание |
|-----------|----------|
| `&Перед("ИмяПроцедуры")` | Выполняется **до** оригинальной процедуры |
| `&После("ИмяПроцедуры")` | Выполняется **после** оригинальной процедуры |
| `&Вместо("ИмяПроцедуры")` | **Заменяет** оригинальную процедуру |
| `&ИзменениеИКонтроль("ИмяПроцедуры")` | Копия с diff-маркерами `#Вставка`/`#Удаление` |

### Diff-маркеры в коде расширения

```bsl
&ИзменениеИКонтроль("ИмяПроцедуры")
Функция Расш_ИмяПроцедуры()
    // ... неизменённый код ...
#Удаление
    СтарыйКод();
#КонецУдаления
#Вставка
    НовыйКод();
#КонецВставки
    // ... неизменённый код ...
КонецФункции
```

## Structure

```
.claude/
  skills/           — навыки для Claude Code (SKILL.md + скрипты)
  docs/             — спецификации XML-форматов 1С
scripts/
  reduce-comparison-report.py      — [1] парсинг отчёта сравнения 1С
  extract-modified-procedures.py   — [2] извлечение процедур в файлы
  generate-extension-procedures.py — [3] генерация _ext.bsl с маркерами
  assemble-extension-modules.py    — [4] сборка модулей с переименованием
  verify-extension-modules.py      — [5] верификация собранных модулей
  deploy-extension-modules.py      — [6] копирование .bsl в расширение
  generate-extension-metadata.py   — [7] генерация XML-метаданных расширения
config.json         — пути к конфигурациям, параметры расширения
work/               — рабочий каталог с извлечёнными процедурами (генерируется)
extension/          — каталог расширения (результат пайплайна)
```

## MANDATORY: Skills-First Rule

**ПЕРЕД написанием скрипта или решения — проверь, есть ли подходящий скилл.**

1. Пользователь просит → проверь список скиллов ниже
2. Скилл есть → используй `Skill` tool, не изобретай заново
3. Скилла нет → пиши своё

## MANDATORY: Autonomy After Approval

**Одна точка согласования на задачу. После «ок» — выполнять автономно.**

- Уточняющие вопросы ПЕРЕД показом плана
- Показать дизайн+план → получить ОДНО одобрение
- После «ок»: НЕ спрашивать «можно продолжить шаг N?»
- Останавливаться только при блокерах

## Task Routing (automatic)

| Сложность | Режим | Что делать |
|-----------|-------|-----------|
| 1-2 объекта, очевидно | **direct** | Скиллами напрямую |
| 3-5 задач, нужен дизайн | **standard** | `/brainstorm` → план → выполнить |
| 6+ задач, архитектурное | **full** | `/brainstorm` → `/write-plan` → `/subagent-dev` |

## Skills

### Расширения конфигурации (CFE)
- `/cfe-init` — создать расширение конфигурации (scaffold)
- `/cfe-borrow` — заимствовать объект из конфигурации в расширение
- `/cfe-patch-method` — перехватить метод: `&Перед`/`&После`/`&ИзменениеИКонтроль`
- `/cfe-diff` — анализ расширения: Mode A (обзор), Mode B (проверка переноса)

### Инспекция и валидация
- `/inspect` — анализ объекта 1С: конфигурация, метаданные, форма, СКД, MXL, подсистема, роль
- `/validate` — валидация: CF, CFE, метаданные, форма, подсистема, интерфейс, роль

### Миграция доработок
- `/migrate-to-extension` — полный пайплайн: отчёт → извлечение → генерация → ИИ-ревью

### Workflow
- `/brainstorm` — идея → дизайн → автономное выполнение (express/standard/full)
- `/write-plan` — создать tasks.md из design.md с атомарными шагами
- `/subagent-dev` — выполнить tasks.md субагентами (implementer + reviewer)

### Session Management
- `/session-save` — сохранить состояние сессии в `session-notes.md`
- `/session-restore` — восстановить сессию и продолжить
- `/session-retro` — ретроспектива: что сработало, что нет, выводы

## Conventions

### 1С кодирование
- Кириллица для кода 1С (BSL), латиница для инфраструктуры
- Табы для отступов в BSL коде
- UTF-8 BOM для PowerShell скриптов с кириллицей

### Скрипты автоматизации
- PowerShell для операций с XML и файловой системой (паттерн из skills)
- Python для парсинга BSL-модулей и извлечения процедур
- Конфигурация путей — через конфиг-файл или аргументы CLI
